---
title: Patient Controller
description: API endpoints for managing patient records in the Nguhöe EHR system
---

The Patient Controller handles all CRUD operations for patient records. Access is restricted to authenticated users with admin, doctor, or receptionist roles.

## Authentication

All endpoints require:
- Middleware: `auth`, `verified`
- Role: `admin`, `doctor`, or `receptionist`

## Endpoints

### List Patients

```http
GET /patients
```

Returns a paginated list of patients with optional search filtering.

#### Query Parameters

<ParamField query="search" type="string" optional>
  Search term to filter patients by full name, document ID, phone, or email
</ParamField>

<ParamField query="page" type="integer" optional default="1">
  Page number for pagination
</ParamField>

#### Response

Returns an Inertia render with:

<ResponseField name="patients" type="object">
  Paginated collection of patient records
  
  <ResponseField name="data" type="array">
    Array of patient objects
    
    <ResponseField name="id" type="integer">
      Patient ID
    </ResponseField>
    
    <ResponseField name="full_name" type="string">
      Patient's full name
    </ResponseField>
    
    <ResponseField name="document_id" type="string">
      Patient's identification document number
    </ResponseField>
    
    <ResponseField name="phone" type="string">
      Contact phone number
    </ResponseField>
    
    <ResponseField name="email" type="string">
      Email address
    </ResponseField>
    
    <ResponseField name="birth_date" type="date">
      Date of birth
    </ResponseField>
    
    <ResponseField name="gender" type="string">
      Gender: `male`, `female`, or `other`
    </ResponseField>
  </ResponseField>
  
  <ResponseField name="current_page" type="integer">
    Current page number
  </ResponseField>
  
  <ResponseField name="per_page" type="integer">
    Items per page (10)
  </ResponseField>
  
  <ResponseField name="total" type="integer">
    Total number of patients
  </ResponseField>
</ResponseField>

<ResponseField name="filters" type="object">
  Active search filters
</ResponseField>

### Create Patient Form

```http
GET /patients/create
```

Displays the form for creating a new patient.

#### Response

Returns an Inertia render of the patient creation form.

---

### Store Patient

```http
POST /patients
```

Creates a new patient record.

#### Request Body

Validated by `StorePatientRequest`:

<ParamField body="full_name" type="string" required>
  Patient's full name (max 255 characters)
</ParamField>

<ParamField body="document_id" type="string" optional>
  Identification document number (max 50 characters, must be unique)
</ParamField>

<ParamField body="birth_date" type="date" optional>
  Date of birth (format: YYYY-MM-DD)
</ParamField>

<ParamField body="gender" type="string" optional>
  Gender: `male`, `female`, or `other`
</ParamField>

<ParamField body="phone" type="string" optional>
  Contact phone number (max 20 characters)
</ParamField>

<ParamField body="email" type="email" optional>
  Email address (max 255 characters)
</ParamField>

<ParamField body="address" type="string" optional>
  Physical address
</ParamField>

<ParamField body="medical_antecedents" type="string" optional>
  Medical history and antecedents
</ParamField>

<ParamField body="allergies" type="string" optional>
  Known allergies
</ParamField>

<ParamField body="chronic_diseases" type="string" optional>
  Chronic diseases and conditions
</ParamField>

<ParamField body="current_medication" type="string" optional>
  Current medications
</ParamField>

<ParamField body="notes" type="string" optional>
  Additional notes
</ParamField>

#### Response

Redirects to `/patients` with success message: "Paciente registrado exitosamente."

### Delete Patient

```http
DELETE /patients/{patient}
```

Soft deletes a patient record. **Only admins can delete patients.**

#### Path Parameters

<ParamField path="patient" type="integer" required>
  Patient ID
</ParamField>

#### Authorization

Returns 403 error if the authenticated user does not have the `admin` role.

#### Response

Redirects back with success message: "Paciente eliminado."

## Implementation Details

- Source: `app/Http/Controllers/PatientController.php`
- Search functionality queries: `full_name`, `document_id`, `phone`, `email`
- Pagination: 10 records per page
- Uses action classes: `CreatePatientAction`, `UpdatePatientAction`
- Soft deletes enabled (records marked as deleted but preserved in database)
