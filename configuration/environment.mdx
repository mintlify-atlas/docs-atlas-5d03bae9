## Overview

Nguhöe EHR uses environment variables to configure application behavior across different environments. All configuration is managed through the `.env` file, which should never be committed to version control.

## Getting Started

Copy the example environment file to create your configuration:

```bash
cp .env.example .env
php artisan key:generate
```

## Application Settings

<ParamField path="APP_NAME" type="string" default="Laravel">
  The name of your application. Used in emails, notifications, and UI elements.
  
  ```bash
  APP_NAME="Nguhöe EHR"
  ```
</ParamField>

<ParamField path="APP_ENV" type="string" default="local">
  The application environment. Determines configuration behavior and error reporting.
  
  **Options:** `local`, `staging`, `production`
  
  ```bash
  APP_ENV=production
  ```
</ParamField>

<ParamField path="APP_KEY" type="string" required>
  Application encryption key. Auto-generated with `php artisan key:generate`. Required for session and data encryption.
  
  ```bash
  APP_KEY=base64:randomgeneratedkeyhere
  ```
</ParamField>

<ParamField path="APP_DEBUG" type="boolean" default="true">
  Enable detailed error messages and stack traces. **Must be `false` in production.**
  
  ```bash
  APP_DEBUG=false  # Production
  APP_DEBUG=true   # Development
  ```
</ParamField>

<ParamField path="APP_URL" type="string" default="http://localhost">
  The base URL of your application. Used for generating links and asset URLs.
  
  ```bash
  APP_URL=https://ehr.example.com
  ```
</ParamField>

<ParamField path="APP_LOCALE" type="string" default="en">
  Default application language.
  
  ```bash
  APP_LOCALE=en
  ```
</ParamField>

<ParamField path="APP_FALLBACK_LOCALE" type="string" default="en">
  Fallback language when the current language is unavailable.
  
  ```bash
  APP_FALLBACK_LOCALE=en
  ```
</ParamField>

<ParamField path="APP_FAKER_LOCALE" type="string" default="en_US">
  Locale for generating fake data in development and testing.
  
  ```bash
  APP_FAKER_LOCALE=en_US
  ```
</ParamField>

<ParamField path="APP_MAINTENANCE_DRIVER" type="string" default="file">
  Storage driver for maintenance mode state.
  
  **Options:** `file`, `database`
  
  ```bash
  APP_MAINTENANCE_DRIVER=file
  ```
</ParamField>

## Security Settings

<ParamField path="BCRYPT_ROUNDS" type="integer" default="12">
  Number of bcrypt hashing rounds for password encryption. Higher values increase security but require more CPU.
  
  **Recommended:** `12` for production
  
  ```bash
  BCRYPT_ROUNDS=12
  ```
</ParamField>

## Database Configuration

<ParamField path="DB_CONNECTION" type="string" default="sqlite">
  Database driver to use.
  
  **Options:** `sqlite`, `mysql`, `mariadb`, `pgsql`, `sqlsrv`
  
  ```bash
  DB_CONNECTION=mysql
  ```
</ParamField>

<ParamField path="DB_HOST" type="string" default="127.0.0.1">
  Database server hostname or IP address.
  
  ```bash
  DB_HOST=127.0.0.1
  ```
</ParamField>

<ParamField path="DB_PORT" type="integer" default="3306">
  Database server port.
  
  **Defaults:** MySQL/MariaDB: `3306`, PostgreSQL: `5432`, SQL Server: `1433`
  
  ```bash
  DB_PORT=3306
  ```
</ParamField>

<ParamField path="DB_DATABASE" type="string" default="laravel">
  Name of the database. For SQLite, this is the path to the database file.
  
  ```bash
  # MySQL/PostgreSQL
  DB_DATABASE=nguhoe_ehr
  
  # SQLite
  DB_DATABASE=/path/to/database/database.sqlite
  ```
</ParamField>

<ParamField path="DB_USERNAME" type="string" default="root">
  Database username for authentication.
  
  ```bash
  DB_USERNAME=nguhoe_user
  ```
</ParamField>

<ParamField path="DB_PASSWORD" type="string">
  Database password for authentication.
  
  ```bash
  DB_PASSWORD=secure_password_here
  ```
</ParamField>

## Session Configuration

<ParamField path="SESSION_DRIVER" type="string" default="database">
  Session storage driver.
  
  **Options:** `file`, `cookie`, `database`, `memcached`, `redis`, `array`
  
  **Recommended:** `database` for production
  
  ```bash
  SESSION_DRIVER=database
  ```
</ParamField>

<ParamField path="SESSION_LIFETIME" type="integer" default="120">
  Session lifetime in minutes. Users will be logged out after this period of inactivity.
  
  ```bash
  SESSION_LIFETIME=120  # 2 hours
  ```
</ParamField>

<ParamField path="SESSION_ENCRYPT" type="boolean" default="false">
  Encrypt session data before storage. Adds security but slight performance overhead.
  
  ```bash
  SESSION_ENCRYPT=false
  ```
</ParamField>

<ParamField path="SESSION_PATH" type="string" default="/">
  Cookie path for session. Usually the root path.
  
  ```bash
  SESSION_PATH=/
  ```
</ParamField>

<ParamField path="SESSION_DOMAIN" type="string">
  Cookie domain for session. Leave null for current domain.
  
  ```bash
  SESSION_DOMAIN=null
  ```
</ParamField>

## Security Considerations

<Warning>
  **Never commit your `.env` file to version control.** It contains sensitive credentials and secrets.
</Warning>

### Production Security Checklist

- Set `APP_DEBUG=false` to prevent information leakage
- Use strong, unique values for `APP_KEY`
- Enable HTTPS and set `APP_URL` to use `https://`
- Use `BCRYPT_ROUNDS=12` or higher for password hashing
- Set appropriate `SESSION_LIFETIME` based on your security requirements
- Use `SESSION_ENCRYPT=true` for sensitive applications
- Restrict database user permissions to only required operations
- Use strong database passwords
- Enable Redis authentication with `REDIS_PASSWORD`
- Set `LOG_LEVEL=error` to reduce disk usage
- Review and remove unused AWS credentials
- Use environment-specific mail settings to prevent accidental emails

### File Permissions

Ensure proper file permissions for the `.env` file:

```bash
chmod 600 .env
chown www-data:www-data .env
```

### Environment Variables in Code

<Note>
  Never use `env()` helper directly in your application code. Always access environment variables through configuration files using `config()`.
</Note>

```php
// ❌ Bad - Direct env() usage
$appName = env('APP_NAME');

// ✅ Good - Use config()
$appName = config('app.name');
```

## Troubleshooting

### Configuration Cache

In production, Laravel caches configuration for performance. After changing `.env`, clear the cache:

```bash
php artisan config:clear
php artisan config:cache
```

### Missing APP_KEY Error

If you see "No application encryption key has been specified":

```bash
php artisan key:generate
```

### Database Connection Issues

Verify database credentials and connectivity:

```bash
php artisan tinker
>>> DB::connection()->getPdo();
```

### Mail Testing

Test mail configuration:

```bash
php artisan tinker
>>> Mail::raw('Test email', function($msg) { $msg->to('test@example.com')->subject('Test'); });
```
